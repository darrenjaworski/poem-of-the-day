## Project Overview

This is a VS Code extension that displays a daily poem in the status bar and a webview panel, fetching from the PoetryDB public API with no authentication required.

## Commands

```bash
npm run compile       # Type-check, lint, and build
npm run watch         # Watch mode (esbuild + tsc in parallel)
npm run package       # Production build (minified)
npm run lint          # ESLint on src/
npm run check-types   # TypeScript type-check only
npm test              # Run VS Code extension tests
npm run vsce:package  # Build and package .vsix
```

To develop: open in VS Code and press F5 to launch the Extension Development Host (uses `.vscode/launch.json`, which runs `npm run watch` as a pre-launch task).

## Architecture

**Entry point:** `src/extension.ts` тАФ all extension logic lives here (394 lines, no other source files).

**Key flow:**
1. Extension activates on `onStartupFinished`
2. Creates a status bar item with a feather icon (ЁЯк╢)
3. Fetches today's poem from PoetryDB, refreshes every 60 minutes
4. User clicking the status bar opens a webview panel with the full poem

**Deterministic daily poem selection:**
- `createDeterministicRandom(seed)` тАФ LCG-based PRNG seeded with the date as `YYYYMMDD`
- `shuffleArray(array, prng)` тАФ Fisher-Yates shuffle using the PRNG
- This guarantees the same poem is shown for a given date across all sessions

**PoetryDB API usage:**
- `GET https://poetrydb.org/author` тАФ list all authors
- `GET https://poetrydb.org/author/{name}` тАФ poems by an author
- No API key required; handled via Node.js built-in `https` module

**Global state:**
- `lastPoemData` тАФ current poem (title, author, body, optional error)
- `currentPoemPanel` тАФ reference to the active `vscode.WebviewPanel`

**Fallback:** If the API fails, a hardcoded default poem (J.V. Cunningham) is displayed.

**Webview HTML** is generated by `getWebviewContent()` using VS Code CSS theme variables for automatic light/dark theme support.

**User setting:** `poemOfTheDay.openOnStartup` (boolean, default `false`) тАФ auto-opens the webview on startup.

## Build System

- **Bundler:** esbuild (`esbuild.js`) тАФ outputs `dist/extension.js` in CJS format with `vscode` as an external
- **Type-check:** `tsc --noEmit` (does not emit files)
- **Tests:** Compiled separately to `out/` by `tsc -p tsconfig.test.json`, run via `@vscode/test-cli`
- `dist/` is excluded from VS Code search/file explorer via `.vscode/settings.json`
